# 프론트엔드 UI/페이지 구현 계획

## 목표
- 랜딩부터 관리자 운영까지 전체 흐름을 웹/태블릿/모바일 대응으로 구현한다.
- 현재 디자인 톤(오렌지 70% + 그린 30%)을 유지하며 일관된 UI 컴포넌트를 확보한다.
- 학습 콘텐츠 상세 흐름은 별도 기획서를 받은 뒤 확정한다.

## 진행 원칙
- 라우팅 중심으로 페이지를 먼저 골격화하고, 이후 데이터 연동을 단계적으로 붙인다.
- 학습 페이지(퀴즈/독해/프로모드 등)는 별도 기획서 수령 전까지 생성하지 않는다.

## 현재 구현 완료
- 랜딩: `/`
- 로그인: `/login`
- 로그인 후 스타트: `/start`
- 관리자 대시보드(요약): `/admin`
- 운영 스테이션: `/ops`

## 단계별 구현 계획

### 1) 인증 영역 (우선)
- 회원가입: `/signup`
  - 기관 선택 포함 (orgs API)
  - 필수 입력: 이름, 지역, 학교, 학년, 학생 전화번호, 학부모 전화번호
  - 학년 기반 초기 레벨 매핑(12레벨)
    - 소쉬르1: 초1, 소쉬르2: 초2, 소쉬르3: 초3
    - 프레게1: 초4, 프레게2: 초5, 프레게3: 초6
    - 러셀1: 중1, 러셀2: 중2, 러셀3: 중3
    - 비트겐슈타인1: 고1, 비트겐슈타인2: 고2, 비트겐슈타인3: 고3
  - 가입 시 진단 테스트 응시 여부 선택
- 비밀번호 재설정: `/reset`

### 2) 학습 영역 (보류)
- 아래 학습 페이지는 **별도 학습 페이지 기획서 수령 후 진행**
  - 일일 퀴즈, 일일 독해, 과제 목록/상세/제출
  - 프로모드/농장모드/글쓰기/시험/대결

### 3) 커뮤니티/상점/결제
- 게시판 목록/상세/작성
- 상품 목록/상세/주문/결제 결과
- 구독 관리

### 4) 관리자 상세
- 기관/반/학생 관리
- 콘텐츠 업로드 및 검수
- 과제/글쓰기 피드백
- 신고/모더레이션
- 플래그 및 시즌/대결 시즌 관리

## 공통 컴포넌트 방향
- 상단 내비게이션, CTA 버튼, 카드형 섹션, 정보 배지
- 반응형 그리드(웹 3열, 태블릿 2열, 모바일 1열)

## 진단/평가 운영 방식
- 진단 테스트 및 각종 평가는 인쇄물로 진행한다.
- 학습자는 종이에 문제를 풀고, OMR 시스템에 답안을 온라인 입력한다.
- 시스템은 입력된 답안을 채점하고 분석 결과를 제공한다.
- 진단 테스트 흐름은 `시험지 출력 페이지 -> OMR 입력 페이지`로 연결한다.
- 시험지 출력 페이지는 A4 세로 PDF를 표시하고, 출력하기 버튼 클릭 후에만 OMR 입력 버튼을 활성화한다.
- 시험지 출력 페이지는 키보드 입력을 차단하고, 테스트 페이지 전반에 캡처 방지(컨텍스트 메뉴/선택/복사 차단 등) 처리를 적용한다.
- 시험지 출력은 서버 프린터 연동 옵션을 제공하고, 설정 시 하드웨어 프린터로 직접 출력되도록 구성한다.
- 출력 페이지는 전체 화면 요청, 포커스 이탈 시 화면 가림, 워터마크 오버레이를 적용한다.

## 대기 사항
