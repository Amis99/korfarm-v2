# 구축 순서 (솔로 빌드)

빠르고 정확하게 출시하기 위한 의존성 기준 순서다. 각 단계는 이전 단계 산출물이 준비된 상태에서 진행한다.

## 1. 뼈대와 운영 기반
- 모노레포 구조 정리, 기본 환경 변수/로깅/에러 규칙/타임존 확정.
- 기능 플래그/긴급 중지 스위치 프레임 구축.
- 파일 업로드(프리사인드)와 공통 응답 포맷 준비.

## 2. 데이터 모델과 계약
- 핵심 테이블 설계(사용자/기관/학습/경제/시즌/대결/과제/게시판/결제).
- JSON 스키마 기준과 업로드 검증 규칙 확정.
- API 계약 스켈레톤 확정.

## 3. 인증/권한
- 회원가입/로그인/권한 조회.
- 역할 기반 접근 제어(HQ_ADMIN/ORG_ADMIN/TEACHER/STUDENT).

## 4. 무료 학습 + 경제 + 시즌
- 일일 퀴즈/독해 조회·제출.
- 씨앗 지급, 인벤토리, 수확 제작, 장부 기록.
- 시즌 랭킹/수상 조회(스냅샷 생성 포함).

## 5. 대결 모드
- 대기실/매칭, 웹소켓 경기, 답안 제출.
- 에스크로 잠금과 정산, 전적 집계, 랭킹.
- 부정행위 탐지와 제재 플로우.

## 6. 유료 학습 + 과제 + 관리자
- 프로/농장별/글쓰기/테스트 창고, 과제 바구니.
- 관리자 모드: 기관/학생/수강반/과제/콘텐츠 업로드/게시판 승인.

## 7. 결제 + 쇼핑몰
- 유료 회원 전환(웹 카드 결제).
- 쇼핑몰 상품/주문/배송, 결제 연동.

## 8. 안정화
- 플래그 온·오프 시나리오 점검.
- 핵심 E2E/부하 테스트와 관측성 알람 점검.
