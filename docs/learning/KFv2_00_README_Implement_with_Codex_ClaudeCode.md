# 국어농장 v2 구현 패킷 README

본 문서는 사용자가 제공한 **학습 페이지 기획 최종안**과 **학습 생성 모듈 지침**을 기준으로, 코덱스 또는 클로드코드에 그대로 전달하여 서비스 버전 구현을 진행할 수 있도록 작성된 문서 묶음의 안내서입니다.

## 1. 문서 묶음 구성

- 00 README  
  이 문서입니다. 전체 파일의 목적, 순서, 작업 방식이 들어 있습니다.

- 01 학습 페이지 엔진 규격서  
  공통 UI와 공통 로직, 타임바, 하이라이트, 모달, 채점, 보상, 벌칙의 표준 동작을 정의합니다.

- 02 학습 콘텐츠 JSON 규격서  
  관리자 입력용 메타 정보 스키마와 학습 유형별 데이터 스키마를 정의합니다.

- 03 관리자 등록 및 운영 규격서  
  본사 관리자, 제휴기관 관리자 권한 규칙, JSON 업로드, 미리보기, 버전 관리, 과제 배정까지 정의합니다.

- 04 백엔드 API 및 DB 규격서  
  SpringBoot Kotlin JDK17, RDS MySQL, S3, Redis 기준으로 엔드포인트와 테이블을 정의합니다.

- 05 학습 엔진 구현 작업 분해서  
  프런트엔드 파일 구조, 컴포넌트 분해, 구현 순서, 테스트 기준을 제공합니다.

## 2. 구현 원칙

### 2.1 서비스 버전 전제
- MVP 단계를 생략하고 즉시 서비스 버전으로 구현합니다.
- 따라서 아래 기능은 초기부터 포함합니다.
  - 무료 학습과 유료 학습 권한 분기
  - 시즌 랭킹, 수상, 수확 제작
  - 관리자 등록, 과제 배정, 리포트 열람

### 2.2 학습 페이지 구현 철학
- 학습 유형이 늘어나도 새 화면을 매번 만들지 않도록 **학습 엔진 1개 + 모듈 플러그인** 구조로 만듭니다.
- 모든 학습은 아래 3개 레이어로 구성합니다.
  - 콘텐츠 데이터 레이어: JSON
  - 엔진 레이어: 하이라이트, 모달, 타임바, 채점, 로그
  - UI 스킨 레이어: 시험지 배경, 지문 박스, 카드 영역, 원고지, 박스 레이아웃

## 3. 코덱스 또는 클로드코드에 전달하는 방법

### 3.1 1회차 프롬프트
- 01, 02, 04 문서를 함께 전달하고 아래 문장을 붙입니다.

> 목표: React 기반의 학습 엔진과 관리자 콘텐츠 등록 기능을 구현한다.  
> 제약: React 19.2.1, JavaScript, SpringBoot Kotlin JDK17, RDS MySQL, S3, Redis.  
> 결과물: 실행 가능한 프런트엔드, 백엔드, DB 마이그레이션, 최소 테스트, 샘플 콘텐츠.

### 3.2 2회차 프롬프트
- 03, 05 문서를 함께 전달하고 아래 문장을 붙입니다.

> 목표: 관리자 권한과 과제 배정, 학습 로그와 리포트, 샘플 시나리오를 연결한다.  
> 결과물: 관리자 페이지에서 JSON 업로드 후 미리보기, 게시, 학생에게 과제 배정, 학생이 수행 후 씨앗 지급까지 종단 동작.

## 4. 산출물 체크리스트

- 학습 엔진 공통
  - 타임바 동작
  - 하이라이트 연출
  - 모달 질문 흐름
  - 정답, 오답 처리 표준화
  - 학습 종료 조건과 결과 요약

- 콘텐츠 등록 공통
  - 메타 정보 입력
  - JSON 업로드 및 스키마 검증
  - 미리보기
  - 게시, 비공개, 종료
  - 버전 관리

- 운영 공통
  - 무료, 유료 권한 분기
  - 시즌 단위 랭킹 집계
  - 수확 제작 규칙과 보상 지급
