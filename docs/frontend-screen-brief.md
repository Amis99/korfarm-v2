# 국어농장 프론트엔드 화면 기획서(디자인 전달용)

## 목적
- 학습 페이지를 제외한 현재 구현 화면의 디자인 제작을 위한 기준 문서입니다.
- 웹/태블릿/모바일 3가지 해상도를 모두 고려합니다.

## 공통 디자인 가이드
- 테마 비율: 오렌지 70% + 그린 30%
- 톤앤매너: 따뜻하고 친근한 농장/학습 분위기, 둥근 형태와 부드러운 그라데이션
- 타이포 권장: 제목은 세련된 라운드 산세리프, 본문은 가독성 높은 산세리프
  - 예: 제목 = Gmarket Sans / ONE Mobile / S-Core Dream, 본문 = Pretendard / SUIT
- 카드/버튼: 둥근 모서리, 은은한 그림자, 강조 CTA는 명확한 대비
- 아이콘/일러스트: 학습과 농장 메타포(씨앗, 수확, 노트, 연필)
- 반응형 기준(예시)
  - Web: 1200px 이상
  - Tablet: 768~1199px
  - Mobile: 360~767px

## 공통 UI 요소
- 상단 네비게이션: 로고, 메뉴, CTA
- 하단 푸터: 회사 정보/문의/정책 링크
- 섹션 카드, 상태 뱃지, KPI 카드
- 입력 폼: 라벨/도움말/에러 상태
- 관리자용 테이블 + 검색/필터

## 화면 목록 및 구성 요소

### 1) 랜딩 (`/`)
- 상단: 로고, 메뉴, CTA 버튼
- 히어로: 2줄 문구 + 서브 카피 + CTA + 메인 이미지
  - 문구: "스스로 완성하는 국어 근육," / "국어농장과 함께하세요."
- 핵심 가치 섹션: 카드 3~6개
- 요금/구독 섹션: 가격 카드 2~3개
- 후기/성과 섹션: 카드 혹은 리스트
- 푸터

### 2) 로그인 (`/login`)
- 좌측(또는 상단) 브랜딩 영역 + 배경 이미지
- 로그인 폼: 아이디(이메일 아님), 비밀번호, 로그인 버튼
- 보조 링크: 회원가입, 비밀번호 찾기

### 3) 회원가입 (`/signup`)
- 좌측 브랜딩 영역(이미지 배경 가능)
- 입력 필드
  - 아이디, 비밀번호
  - 이름, 지역(대한민국 광역시/도 드롭다운), 학교
  - 학년(초1~고3) 선택 → 레벨 매핑
  - 학생 전화번호, 학부모 전화번호(3-4-4 자동 하이픈)
  - 진단 테스트 응시 여부 선택
- 하단 보조 링크: 로그인

### 4) 비밀번호 재설정 (`/reset`)
- 이메일 대신 아이디 기준 안내 문구(간단한 폼)

### 5) 스타트 페이지 (`/start`)
- 상단: 사용자 인사, 레벨/XP/목표 카드
- 오늘의 학습 카드: 일일 퀴즈, 일일 독해 등
- 과제 바구니 섹션
- 하단 탭바(홈/학습/커뮤니티/상점 등)

### 6) 커뮤니티
- 목록 (`/community`): 게시글 카드 리스트 + 필터/검색 + 글쓰기 CTA
- 상세 (`/community/post/:postId`): 본문, 첨부, 댓글
- 작성 (`/community/new`): 제목/내용/첨부/등록

### 7) 상점/결제
- 상점 목록 (`/shop`): 상품 카드 그리드 + 필터/정렬
- 상품 상세 (`/shop/products/:productId`): 이미지/설명/가격/구매 CTA
- 구독 (`/subscription`): 구독 상태, 갱신 CTA
- 결제 결과 (`/payment/result`): 성공/실패 상태 카드

### 8) 운영 콘솔 (`/ops`)
- API Base, Token 입력
- 프로필/구독 상태 카드
- API 액션 리스트(버튼)
- WebSocket 콘솔 로그

### 9) 관리자 대시보드 (`/admin`)
- KPI 카드
- 최근 가입자/처리 대기 테이블
- 알림/플래그 카드

### 10) 관리자 상세 목록
- 기관 (`/admin/orgs`)
- 반 (`/admin/classes`)
- 학생 (`/admin/students`)
- 콘텐츠 (`/admin/content`)
- 과제/필기 (`/admin/assignments`)
- 시즌 (`/admin/seasons`)
- 상점 상품 (`/admin/shop/products`)
- 주문 (`/admin/shop/orders`)
- 결제 (`/admin/payments`)
- 신고 (`/admin/reports`)
- 플래그 (`/admin/flags`)

각 화면 구성
- 상단: 검색/필터
- 본문: 데이터 테이블 + 상태 뱃지
- 요약 카드(선택)

### 11) 진단 테스트 출력 (`/diagnostic/print`)
- PDF 뷰어(프린트용 A4 세로)
- 버튼 3개만 배치: 출력하기 / 답안 제출하기 / 다음에 응시하기
- 출력하기 후에 답안 제출 버튼 활성화

### 12) OMR 입력 (`/omr`)
- 20문항 리스트
- 객관식 선택지는 원문자 ①~⑤
- 제출 버튼

## 보류 항목
- 학습 페이지(커리큘럼/문제/학습 콘텐츠)는 별도 기획서 수신 후 제작 예정
