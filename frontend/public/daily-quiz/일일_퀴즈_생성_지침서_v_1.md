# 12레벨 일일 퀴즈 생성 지침서

이 문서는 소쉬르1부터 비트겐슈타인3까지 12레벨에 대해, 매일 10문항으로 구성된 퀴즈 세트를 365일치 자동 생성하기 위한 에이전트용 지침입니다.

## 1. 목표와 기본 원칙

1. 각 레벨은 초등 1학년부터 고등 3학년 수준까지를 12단계로 세분화합니다.
2. 하루 1세트는 10문항으로 구성합니다.
3. 10문항은 국어 10대 역량에 대응하되, 메타인지는 제외하고 문법 또는 어법을 포함합니다.
4. 중심 낱말 찾기, 중심 문장 찾기 유형은 금지합니다.
5. 요약문 만들기 유형은 허용합니다.
6. 서술형 출제가 가능합니다.
7. 모든 문항은 정답의 유일성이 보장되어야 합니다.
8. 모든 지문과 문항은 저작권 문제를 피하기 위해 반드시 새로 창작합니다.
9. 수능 기출 주제는 참고하되, 기출 문장이나 구성의 복제를 금지합니다.
10. 문항과 선택지에 괄호 속 부연을 사용하지 않습니다.

## 2. 레벨 체계와 난이도 기준

다음은 레벨을 학년대와 난이도 축으로 매핑한 운영 기준입니다.

- 소쉬르1: 초등 1학년
- 소쉬르2: 초등 2학년
- 소쉬르3: 초등 3학년
- 프레게1: 초등 4학년
- 프레게2: 초등 5학년에서 6학년
- 프레게3: 중등 1학년
- 러셀1: 중등 2학년
- 러셀2: 중등 3학년
- 러셀3: 고등 1학년
- 비트겐슈타인1: 고등 2학년
- 비트겐슈타인2: 고등 3학년
- 비트겐슈타인3: 수능 상위권 목표 난도

지문 길이 기준은 문항 구성에서 자동 조절합니다.

- 소쉬르1에서 소쉬르3: 2문단 이하, 3문장에서 8문장
- 프레게1에서 프레게2: 2문단에서 3문단, 8문장에서 16문장
- 프레게3에서 러셀2: 3문단 내외, 14문장에서 24문장
- 러셀3에서 비트겐슈타인2: 3문단에서 5문단, 20문장에서 32문장
- 비트겐슈타인3: 4문단에서 6문단, 28문장에서 40문장

## 3. 10대 역량과 10문항 슬롯 설계

하루 10문항은 아래 슬롯을 고정으로 사용합니다. 매일 동일한 순서를 유지하는 것을 원칙으로 하되, 서비스 설계상 섞을 필요가 있으면 내부 표시만 유지하고 출력 순서만 바꿀 수 있습니다.

1. 어휘력
2. 문장 독해력
3. 구조 독해력
4. 배경지식
5. 추론
6. 비판적 사고
7. 창의적 사고
8. 문제 해결
9. 표현력
10. 문법 또는 어법

각 슬롯은 다음 조건을 따릅니다.

- 한 문항은 하나의 역량을 주역량으로 갖습니다.
- 다른 역량이 섞여도 되지만, 채점의 핵심은 주역량에 있어야 합니다.
- 문항의 근거는 지문 또는 보기 또는 조건에 명확히 있어야 합니다.
- 배경지식 문항은 학년 수준의 교과 지식 또는 상식 범주에서만 요구합니다.

## 4. 지문 구성 규칙

### 4.1 지문 유형

하루 10문항 중 최소 6문항은 지문 기반으로 구성합니다.

- 설명문, 논설문, 보고서형 글, 안내문, 대화문, 발표문, 생활문
- 문학 지문은 프레게3부터 적극 사용합니다.

### 4.2 지문 분배

- 소쉬르1부터 프레게2: 교과 연계 정보 지문 비중을 높입니다.
- 프레게3부터 비트겐슈타인3: 수능형 독서 지문과 문학 지문 비중을 높입니다.

### 4.3 금지 요소

- 특정 작품 원문을 재현하지 않습니다.
- 실제 수능 지문의 문장 구조를 그대로 따라 하지 않습니다.
- 중심 낱말 찾기, 중심 문장 찾기라는 표현과 그에 준하는 요구를 만들지 않습니다.

### 4.4 예시문과 발문의 분리 (지문 필드 활용)

문항에서 분석 대상이 되는 문장이나 짧은 글은 반드시 `stem`이 아닌 `passage` 필드로 분리합니다. 발문(`stem`)에는 묻는 내용만 담고, 대상 텍스트는 지문(`passage`)으로 처리합니다.

- **나쁜 예**: `stem`: "다음 문장의 주어는 무엇인가요? \"아름다운 꽃이 정원에 피었습니다.\""
- **좋은 예**: 
  - `stem`: "다음 문장의 주어는 무엇인가요?"
  - `passage`: "아름다운 꽃이 정원에 피었습니다."

한 문장짜리 짧은 예시라도 분석 대상이라면 지문으로 취급합니다.

## 5. 레벨별 소재와 주제 운영 규칙

### 5.1 소쉬르1부터 프레게2: 초등과 중등 교과 소재 적극 포함

이 구간은 사회, 과학, 도덕, 실과, 미술, 음악, 체육 등 교과 내용을 적극 반영합니다. 다만 국어 문항이므로 지식 자체를 시험하기보다, 그 지식을 담은 짧은 글을 이해하고 활용하게 해야 합니다.

권장 소재 예시는 다음과 같습니다.

- 과학: 물의 상태 변화, 계절과 날씨, 간단한 실험 절차, 생물의 특징, 생태계 기초, 에너지 절약
- 사회: 우리 동네 공공기관, 규칙과 약속, 지도 읽기 기초, 지역의 특징, 기본 경제 생활, 환경과 공동체
- 도덕과 생활: 배려, 공정, 책임, 갈등 해결
- 실과와 기술 기초: 안전, 간단한 도구 사용, 재활용

문항 설계 원칙은 다음과 같습니다.

- 필요한 배경지식은 지문 속에 설명을 포함하거나, 학년 수준에서 당연히 아는 상식 범위만 사용합니다.
- 어려운 용어는 지문 안에서 쉬운 말로 풀어서 정의합니다.

### 5.2 프레게3부터 비트겐슈타인3: 수능 독서와 문학의 주제와 예상 주제 반영

이 구간은 실제 수능 독서와 문학에서 반복적으로 다루는 주제 축을 기반으로 소재를 선정합니다. 다음 원칙을 지킵니다.

- 기출 주제는 참고하되, 지문은 반드시 새로 창작합니다.
- 수능형 독서 지문은 정의, 분류, 비교, 인과, 사례 적용, 반론 제시 같은 구조를 갖도록 설계합니다.
- 문학 지문은 작품을 직접 인용하지 않고, 장르의 관습과 표현 특성을 반영한 창작 텍스트로 구성합니다.

수능형 주제 축 예시는 다음과 같습니다.

- 인문: 인격 동일성, 자아 개념, 미학과 예술 경험, 언어 철학, 인식론
- 사회: 법 해석 방법, 계약과 책임, 경제 제도와 시장 실패, 사회 제도와 규범
- 과학과 기술: 기계 학습의 개념, 확산모델과 생성 기술의 원리, 정보 처리와 데이터, 기후와 생태 변화
- 언어: 문자 체계, 음운과 표기, 맞춤법 규정 적용, 담화와 매체
- 문학: 고전 설화와 판소리계 서사, 현대시의 정서와 이미지, 소설의 인물과 갈등, 서술 방식의 효과

예상 주제는 다음 성격으로 확장합니다.

- 인공지능 윤리와 저작권
- 플랫폼 경제와 노동
- 기후 위기와 생물다양성
- 생명과학과 개인정보
- 디지털 매체 환경의 담화

## 6. 슬롯별 문항 템플릿과 난이도 조절

각 슬롯은 아래 템플릿 중에서 레벨에 맞게 선택합니다. 동일 슬롯이라도 날마다 템플릿을 바꿔 반복감을 줄입니다.

### 6.1 어휘력

- 문맥 속 의미 고르기
- 비슷한 뜻 또는 반대 뜻 고르기
- 다의어 의미 구분
- 적절한 어휘 선택하여 문장 완성

난이도 조절

- 저학년: 생활 어휘 중심, 보기 단순화
- 고학년: 한자어, 추상어 포함, 문맥 단서 활용 요구

### 6.2 문장 독해력

금지 사항을 피하면서 문장 수준 처리를 평가합니다.

- 지시어가 가리키는 대상 찾기
- 문장 간 관계 고르기, 예를 들면 원인과 결과, 대조, 예시
- 문장 성분 변화에 따른 의미 차이 비교

### 6.3 구조 독해력

중심 문장 찾기 없이 구조 파악을 평가합니다.

- 글의 전개 방식 고르기, 예를 들면 문제 해결, 비교 대조, 시간 순서
- 문단의 기능 고르기, 예를 들면 정의 제시, 사례 제시, 반론 제시
- 글의 목적 또는 글쓴이의 의도 판단

### 6.4 배경지식

- 교과 연계 개념을 지문으로 제시한 뒤, 지문 내용과 연결되는 사실 판단
- 동일 개념의 새로운 사례 적용

주의

- 프레게3 이상에서는 배경지식이 아니라 독서형 설명 지문을 통해 개념을 주고 판단시키는 방식이 원칙입니다.

### 6.5 추론

- 생략된 전제 보완
- 조건을 바탕으로 결론 도출
- 인물의 의도나 태도 추론

### 6.6 비판적 사고

- 주장과 근거의 타당성 판단
- 반례 제시가 가능한지 판단
- 개념 혼동이나 논리적 비약 찾기

### 6.7 창의적 사고

정답 유일성을 확보할 수 있는 형태로만 출제합니다.

- 제시된 조건을 만족하는 새로운 예시 만들기
- 두 개념을 연결하는 적절한 비유 선택
- 문장 변환 과제를 제시하고 정해진 기준에 가장 부합하는 선택지 고르기

서술형을 사용할 때는 채점 기준을 명확히 제공합니다.

### 6.8 문제 해결

- 규칙 기반 문제, 예를 들면 표의 규칙 읽기, 간단한 분류 규칙 적용
- 조건 조합에 따라 가능한 경우를 판별

### 6.9 표현력 (서술형 문장 완성 연습)

서술형 문장 만들기 연습을 10번 슬롯의 '표현력' 전용 유형으로 고정하여 운영합니다.

- **형식**: 완성할 문장의 빈칸을 순차적으로 채우는 '서술형 연습' 모달 인터랙션.
- **구성**:
  1. 지문과 문제를 제시합니다.
  2. 완성해야 할 문장의 전체 구조를 빈칸(블록)으로 보여줍니다.
  3. 첫 어절부터 마지막 어절까지 순차적으로 4지 선다(정답 1개, 오답 3개) 중에서 올바른 표현을 고르게 합니다.
  4. 예: "카는 사실을..." 문장을 만들 때, [카는 / 랑케는 / 콜링우드는 / 크로체는] -> [해석을 / 사실을 / 기록을 / 상상을] 순으로 선택.
- **목표**: 글의 핵심 내용을 담은 문장을 정확한 어휘와 문장 성분 호응에 맞게 완성하는 능력을 평가.

주의

- 중심 문장 찾기와 유사한 표현을 쓰지 않습니다.
- 요약은 내용의 포괄성과 정확성을 기준으로 평가합니다.

### 6.10 문법 또는 어법

레벨별 범위를 고정해 과도한 난도 상승을 막습니다.

- 소쉬르1부터 소쉬르3: 문장 부호, 높임 표현 기초, 조사 사용, 간단한 띄어쓰기
- 프레게1부터 프레게2: 품사 기초, 시제와 높임 기초, 문장 성분 기초, 맞춤법 기초
- 프레게3부터 러셀2: 문장 성분 확장, 사동과 피동, 어휘와 문법의 결합, 표준 발음의 기초
- 러셀3부터 비트겐슈타인2: 담화 표지, 문장 종결 표현, 어법 오류 탐지, 규정 적용형 맞춤법
- 비트겐슈타인3: 고난도 규정 적용, 음운 변동과 표기, 중세 국어 기초 지식의 활용형, 매체 언어

## 7. 수능형 소재를 일일 퀴즈에 녹이는 방식

프레게3 이상은 다음과 같은 수능형 구성 요소를 주기적으로 포함합니다.

- 정의와 분류
- 개념의 역사적 변화
- 법 조문의 문리 해석과 체계적 해석의 대비
- 인격 동일성 같은 철학적 논변의 비교
- 기후와 생태 환경에서 관측 자료를 제시하고 해석
- 문자 체계와 표기의 변화

주의

- 실제 수능 지문 문장을 재현하지 않습니다.
- 수능의 발문 표현을 그대로 복제하지 않습니다.
- 다만 수능에서 요구하는 사고 유형을 동일하게 구현하는 것은 허용합니다.

## 8. 연간 로테이션 규칙

### 8.1 주제 로테이션

연간 365세트에서 주제 반복을 줄이기 위해 7일 단위로 큰 축을 순환합니다.

- 1일차: 과학
- 2일차: 사회
- 3일차: 인문
- 4일차: 기술
- 5일차: 예술과 문화
- 6일차: 언어와 규범
- 7일차: 혼합형

초등 구간에서는 과학과 사회 비중을 늘리고, 고등 구간에서는 인문과 사회 비중을 늘립니다.

### 8.2 지문 장르 로테이션

프레게3 이상은 다음 장르 비율을 유지합니다.

- 독서형 지문: 60퍼센트
- 문학형 지문: 30퍼센트
- 매체와 화법형 지문: 10퍼센트

초등 구간은 생활문과 설명문 중심으로 운영합니다.

## 9. 출력 형식 규격

에이전트는 하루 세트마다 다음 두 가지 출력물을 동시에 생성합니다.

1. 학생용
- 정답과 해설이 없는 버전

2. 운영용
- 정답, 해설, 서술형 채점 기준을 포함한 버전

권장 데이터 구조는 다음과 같습니다.

```json
{
  "level": "프레게3",
  "date": "2026-02-03",
  "set_id": "F3-20260203",
  "items": [
    {
      "no": 1,
      "competency": "어휘력",
      "format": "객관식",
      "passage": "",
      "question": "...",
      "choices": ["...", "...", "...", "...", "..."],
      "answer": "3",
      "explanation": "..."
    }
  ]
}
```

주의

- 학생용에서는 answer와 explanation을 제거합니다.
- 서술형은 answer에 모범 답안을, explanation에 채점 기준을 함께 포함합니다.

## 10. 품질 검사 체크리스트

에이전트는 매 세트 생성 후 아래를 자동 점검합니다.

1. 10문항이 모두 서로 다른 주역량을 갖는지 확인
2. 중심 낱말 찾기, 중심 문장 찾기 유형이 포함되지 않았는지 확인
3. 각 객관식 문항의 정답이 하나로 고정되는지 확인

7. 오답 선택지가 과도하게 명백하지 않은지 확인
8. 표현력과 문법 문항에서 규정 또는 기준이 명확한지 확인
